## Code Review

For my code review, I selected my **CS 465 full-stack artifact**, which originally was **Travlr Getaways**. I chose this artifact because it already included a customer-facing Express application, an Angular admin application, a REST API, and a MongoDB database. That gave me a strong base to analyze for structure, logic, security, and future enhancement opportunities.

In the code review, I walked through the existing functionality of the project, including how the public-facing and admin-facing portions were separated, how the API was structured, and how the application handled CRUD operations and authentication. I also identified several areas for improvement. The main issues I focused on were that the application was still tied to a travel-management use case, the data model was not yet aligned with a more realistic operational workflow, and the artifact had room for stronger logic, better database structure, and more security-minded design choices.

The code review also served as the planning foundation for the three enhancements in this portfolio. For software design and engineering, I planned to repurpose the application into **SecureRastrear**, an internal incident-tracking system. For algorithms and data structures, I planned to add prioritization, sorting, searching, and filtering logic so the system could behave more like a triage tool. For databases, I planned to strengthen the MongoDB layer through better incident schema design and record tracking. That review process helped me think about the artifact less as a class project and more as a system that could be iteratively improved across multiple technical categories.
