## Code Review

I decided to conduct the code review of my **CS 465 full-stack artifact**. Initially, it was called **Travlr Getaways**. I picked this particular assignment since it already had a customer-oriented Express application, an Angular administration tool, REST API and a MongoDB database implemented. It means that I had good ground to analyze the structure of this application, its logic, the way it works from a security perspective, and where there can be improvements made in the future.

During the code review, I reviewed the basic functionality of the project. This included separation between public and admin parts of the app, RESTful API organization, handling of CRUD operations, and authentication system of the app. Several points for improvements have been mentioned. In particular, the main problem with this code is the fact that it still uses a travel management use case, data model is far from being operational workflow-like, and logic of the artifact, its database organization, and its security require improvement.

The code review also served as the planning foundation for the three enhancements in this portfolio. For software design and engineering, I planned to repurpose the application into **SecureRastrear**, an internal incident-tracking system. For algorithms and data structures, I planned to add prioritization, sorting, searching, and filtering logic so the system could behave more like a triage tool. For databases, I planned to strengthen the MongoDB layer through better incident schema design and record tracking. That review process helped me think about the artifact less as a class project and more as a system that could be iteratively improved across multiple technical categories.
