## Enhancement 2: Algorithms and Data Structures

### Artifact Description

The artifact for this enhancement is the same **CS 465 full-stack application** that I repurposed into **SecureRastrear**. By continuing to use the same artifact, I was able to build on one coherent system instead of splitting my focus across multiple unrelated projects.

### Why I Included This Artifact

I selected this artifact again because it gave me a realistic way to demonstrate algorithms and data structures in a working application rather than as an isolated exercise. Once the artifact became an incident-tracking system, it made sense to push it further into a triage-oriented workflow that could sort, prioritize, and organize incidents more intelligently.

### Enhancement Implemented

For this enhancement, I added **priority scoring**, **days-open calculation**, **searching**, **filtering**, **sorting**, and **summary metrics** to the incident list. The main goal was to turn the record display from a basic CRUD list into something more like a decision-support tool. Instead of simply listing incidents, the application now evaluates them based on structured logic and helps the user identify more urgent records first.

### Skills Demonstrated

This enhancement demonstrated my ability to:
- apply algorithmic thinking to a practical problem
- design weighted logic to rank and prioritize records
- use structured filtering and sorting to improve usability
- work with collections of records in a way that supports decision-making

### Course Outcome Alignment

This enhancement most strongly supports:
- **Outcome 3**, because it applies algorithmic principles to solve a more realistic operational problem
- **Outcome 4**, because it uses practical tools and logic to create a more useful system
- **Outcome 2**, because the enhancement had to be clearly communicated through both the interface and the portfolio narrative

### Reflection

One of the biggest things I learned from this enhancement was that useful algorithms do not always have to be complicated to matter. In this case, a relatively simple weighted scoring system added a lot of value because it changed how the data could be used. I also learned more about how logic and usability connect. It is one thing to compute a score, but it is another to make that score meaningful in the interface through sorting, filters, and summary information. The main challenge here was staying focused on the category and not overbuilding the project into something much bigger than the milestone required.
