## Enhancement 1: Software Design and Engineering

### Artifact Description

The artifact I selected for this enhancement is my full-stack **Travlr Getaways** project from **CS 465 Full Stack Development I**, originally created in 2025. The project included an Express customer-facing site, an Angular admin application, a REST API, and a MongoDB database.

### Why I Included This Artifact

I chose this artifact because it had the strongest software engineering foundation out of the projects I considered. It already had real separation between the customer site, the admin side, the API layer, and the database model. That gave me room to demonstrate more than cosmetic changes. It also fit well with my career direction because it could be repurposed into something more practical and more aligned with secure operational systems.

### Enhancement Implemented

For this enhancement, I repurposed the project into the first version of **SecureRastrear**, a more professional internal incident-tracking system. I changed the visible branding from a travel application to an incident-management workflow, updated the admin interface to use incident language instead of trip language, and added new structured fields such as **status** and **severity**. I preserved the overall architecture and authentication flow while improving how the system presented and handled records.

### Skills Demonstrated

This enhancement demonstrated my ability to:
- adapt an existing full-stack application to a new use case
- improve the structure and maintainability of a software system
- preserve a working architecture while implementing meaningful change
- improve the relevance of a user workflow without rebuilding the project from scratch

### Course Outcome Alignment

This enhancement most strongly supports:
- **Outcome 2**, because it required clear technical communication and professional presentation
- **Outcome 4**, because it shows practical use of tools and techniques to create a more valuable system
- **Outcome 5**, because it preserved authentication and moved the project in a more security-minded direction

### Reflection

One of the biggest things I learned from this enhancement was how different it feels to improve an existing system instead of starting fresh. I had to think more about compatibility, scope, and what changes would deliver the most value without breaking the project. One challenge was that the original artifact was built around travel data, so I had to carefully reshape the workflow into an incident system while keeping the code stable. This enhancement helped me understand software engineering more as a process of structured improvement and less as simply writing code from scratch.
